这是一个完全由 AI 辅助生成的纯前端 Todo List 产品。从空仓库开始，实现了零配置、开箱即用的极简体验。

## 💡 这一次 AI 帮你跨过的 1 个最大坑

**最大的坑是：直接操作 DOM 导致的状态管理混乱与数据丢失。**

在传统的前端新手思维里，添加一条待办事项就是 `document.createElement` 插入一行 HTML，删除就是从 DOM 树里 `remove`。但这样做在引入 `localStorage` 时是个灾难——界面的显示状态和本地存储里的数据经常对不上号，修改文本时甚至不知道该去更新数组里的哪一条。

这一次 AI 帮我跨过这个坑，强行给我引入了 **“数据驱动 (Data-Driven)”** 的思想：
整个应用只维护一个核心数组 `tasks`。所有的操作（增、删、改、状态切换）都**绝对禁止直接修改页面元素**，而是只去修改这个 `tasks` 数组。修改完毕后，统一调用一次 `renderTasks()` 函数，让页面根据最新的数组重新生成。这彻底解耦了 UI 和数据，让 `localStorage` 的存取变得极其简单且不会出现 Bug。
